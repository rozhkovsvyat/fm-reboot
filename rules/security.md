# Security (fm-reboot)

Действует ВСЕГДА. При конфликте с другими правилами — security побеждает.

## 1. Детекция секретов — немедленный STOP

Остановись и предупреди при обнаружении паттернов:
`password=`, `token=`, `api_key=`, `secret=`, `Bearer `, `sk-`, `ghp_`, `glpat-`, `xox`,
`AKIA`, `-----BEGIN` (private key), connection strings с креденшелами, JWT (`eyJ...`).

```
CREDENTIAL DETECTED
Тип    : [API key / пароль / токен / сертификат]
Действие: Не обрабатываю и НЕ пишу в vault. Удали секрет — продолжим.
          Если уже утёк — ротируй немедленно, считай скомпрометированным.
```

## 2. Секретные файлы — НЕ читать проактивно

`.env`, `.env.*`, `secrets/`, `*.key`, `*.pem`, `*.p12`, `id_rsa`, `~/.aws/credentials`,
`~/.claude/yougile-token`, любые `*-token`. Доступ — только по явному запросу пользователя.
Нашёл секрет в файле — предупреди, НЕ включай в вывод.

## 3. ПДн / личные данные

Не извлекать, не агрегировать, не экспортировать ПДн. В примерах — вымышленные данные
(`Иванов А.С.`, `+7-555-01-00`). При выводе маскировать: `j***@example.com`.

## 4. Безопасность кода (короткий OWASP-чек)

- **Injection** — параметризованные запросы, экранирование; `eval()`/`exec()`/динамический `require` — запрещены.
- **Секреты** — не хардкодить, читать из env/секрет-стора; HTTPS везде; пароли через bcrypt/argon2.
- **Access control** — проверка прав на каждом endpoint; CORS не `*` в проде.
- **Логи** — НЕ логировать секреты/ПДн/sensitive payload.
- **Зависимости** — перед добавлением проверь поддержку/дату релиза/CVE.

## 5. Разрушительные / необратимые операции — подтверждение пользователя

Спрашивай ДО выполнения: `git push`, force-push, удаление файлов/данных, `DROP/TRUNCATE/DELETE`,
прод-операции, изменения CI/CD, установка зависимостей, деплой. Подтверждение фиксируй:
`<операция> — подтверждено пользователем: YYYY-MM-DD`.

## 6. Verify external invariants before hardcode

Перед записью значения, пришедшего НЕ от человека одним сообщением (endpoint, bucket, id доски,
имя поля внешнего API) в shared-state (конфиг, vault, код) — **верифицируй** (probe / docs / спроси
источник). Не хардкодь по памяти. (YouGile API уже выверен в `_system/tools/yougile-sync/README.md`.)
