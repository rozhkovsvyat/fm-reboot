# Task discipline (YouGile, доска reboot)

Трекинг задач проекта — в YouGile, не в файлах vault'а. CLI и настройка — `_system/tools/yougile-sync/`.

## Intent-триггеры (распознавай из свободной формулировки)

| Что говорит пользователь | Действие |
|---|---|
| «создай задачу», «поставь таск», «закинь в reboot» | `create` / `/create-task` |
| «какие задачи», «что в работе», «мой бэклог» | `list` |
| «возьми X в работу» | `move --column-name doing` + предложи |
| «задача готова», «закрой таск» | `done` или `move --column-name done` + предложи |
| «прокомментируй задачу» | `comment` |
| «покажи задачу» | `show` |

## Правила

1. **Не меняй статус молча.** Перед `move`/`done` — подтверди: «Задача "X" → done?».
2. **После значимой работы по задаче** — предложи обновить её статус ИЛИ добавь коммент с прогрессом.
   Если в коммитах есть привязка к задаче — это сигнал, что она в работе.
3. **Создание задачи** — заполни осмысленно: title + (по возможности) описание + дедлайн.
   Дедлайн неясен — не выдумывай, спроси.
4. **Комментарий в задачу** — короткий апдейт (3-8 строк). Длинный отчёт → положи в vault
   (`drafts/`, `state/...`), в коммент дай TL;DR + путь к файлу.
5. **Связь vault ↔ YouGile.** Развёрнутый контекст (дизайн, разбор) живёт в vault'е; YouGile держит
   статус и короткие апдейты. Не дублируй полотна в обе стороны.

## Хук

`check-yougile-tasks.ps1`: SessionStart — сводка (`YOUGILE:`), промпт/Stop — дельта
(`YOUGILE-UPD:`/`YOUGILE-EOT:`). Без токена (`~/.claude/yougile-token` / env `YOUGILE_TOKEN`) — молчит.

## Безопасность

Токен YouGile — только в `~/.claude/yougile-token` или env. Никогда не коммить, не печатать в чат,
не использовать `--debug`-флаги, печатающие авторизацию.
