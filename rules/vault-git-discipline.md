# Vault git discipline

Vault шерится между машинами (и, когда появится второй человек — между людьми) через Obsidian Git
plugin с авто-pull/commit/push. История линейная: коммиты вида `vault backup: <дата>`.

## Правила

1. **Sync first.** Перед чтением/правкой vault'а — `git pull --ff-only` (хук SessionStart уже делает; вручную не дублируй).
2. **`--ff-only` упал → СТОП.** Локальные коммиты разошлись с remote / конфликт — **остановись и сообщи пользователю**. НЕ делай `--rebase`, `merge`, `--force` автономно: линейная история важнее тихого слияния.
3. **Авто-коммит делает плагин.** Вручную `git commit`/`push` в vault — только если пользователь явно попросил. Гонки с Obsidian Git → конфликты.
4. **Append-only журналы** (`inbox/*`, `state/activity.md`, `state/daily/*`, `state/weekly/*`) при конфликте сливай **three-way вручную** (объединяй обе стороны), не выбирай одну версию — иначе теряешь чужие записи.
5. **Git push кода** (в связанных репо, не vault) — разрушительная операция, спрашивай подтверждение (см. [[rules/security]] §5).

## Подключение второго человека / своего код-репозитория

- Vault → личный GitHub (private repo). Obsidian Git настраивается на этот remote.
- Второй человек клонирует тот же репо, ставит `git config kb.author <handle>` (или setup.ps1 определит по email из `team/_handles.yml`).
- Код-репозиторий проекта можно подключить к хукам vault'а через `_system/tools/vault-link.ps1` (хуки vault начнут работать и из cwd код-репо).
