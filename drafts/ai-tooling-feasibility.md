> Связано: [[reboot-context-intake]] → «Идеи / направления», [[../decisions/0002-release-as-singles]].

## Как я (Claude) работаю с аудио — честно (вводная для обоих направлений)

- **Нативно «слушать» и судить о качестве звука — НЕ умею.** Есть зрение (картинки/спектрограммы), слуха нет.
  «Звучит ли красиво/профессионально» на слух не оценю.
- **Через код — умею многое:**
  - *Анализ:* LUFS/громкость, спектральный баланс, RMS/пик/крест-фактор, клиппинг, сибилянты,
    стерео-ширина, маскирование вокал↔бит по частотам, темп, оценка тональности, спектрограммы.
  - *Обработка:* EQ, гейн, нормализация, грубый де-эссинг, компрессия, питч/тайм (rubberband/librosa), рендер.
  - *Стек:* Python (librosa, numpy, scipy, soundfile, pyloudnorm, pydub) + ffmpeg. Спектрограммы строю как
    картинки и «вижу» их.
- **Вывод:** мои «уши» = метрики + картинки, не восприятие. Любой инструмент сведения будет
  метрико-эвристическим; финальные ~20% (вкус, «ёкнуло») — твой слух.

## (A) Агент для сведения в DAW

DAW: **Reaper >> FL Studio.** Reaper = полный ReaScript API (Lua/Python): дорожки, FX, параметры,
рендер. FL закрыт (MIDI-scripting узкий; `.flp` проприетарный, PyFLP читает / частично пишет).

| Тир | Что | Реально? | Срок (part-time) | Качество |
|---|---|---|---|---|
| 0 | Я-консультант: цепочка + настройки на трек, ты делаешь руками | ✅ сейчас | 0 | зависит от тебя |
| 1 | Скрипт-анализатор стемов → диагностика + рекомендованные настройки (DAW не трогает) | ✅ | ~неск. дней | польза высокая, риск низкий |
| 2 | ReaScript ставит базовую цепочку из Тира-1 и рендерит черновик | ✅ муторно | ~2–4 недели | ~70-80%, дальше слухом |
| 3 | Агентный цикл рендер→метрики/критика→правка (MCP над ReaScript) | ⚠️ амбициозно | ~1–3 мес | R&D, без гарантии |
| 4 | Автономные профи-миксы «по слуху» | ❌ нет | — | — |

## (B) Интеграция с Suno

- **Концептуально — самое лёгкое и потенциально самое полезное** из идей: если есть API, это просто
  HTTP-вызовы (нет «перцептивного» барьера, как в сведении). Из чата я мог бы: собирать/оптимизировать
  промпты, **батч-генерить вариации** (ваш цикл «мучаем генерацию» — автоматизировать), тянуть
  результаты/стемы, систематизировать отбор.
- **Барьер — доступ, не способность.** На момент моих знаний у Suno **нет открытого официального
  публичного API** (генерация — через веб-приложение). Есть неофициальные обёртки (reverse-engineered,
  по cookie/сессии) и сторонние врапперы — **серые, ломкие, риск бана аккаунта, против ToS**.
- ⚠️ **Могло измениться** — доступ к API у AI-сервисов меняется быстро. Нужна свежая проверка. Готов сделать ресёрч.
- **Без API уже сегодня:** «человек-в-цикле» — я генерю оптимальные промпты + структуру + правки текста,
  ты вставляешь в Suno. Нулевая интеграция, польза сразу.

## Приоритет (как «вторая половина»)

🔴 **Текущее узкое место — сведение/мастеринг и выпуск первого сингла, а НЕ генерация.** Suno у вас и так
работает. Оба инструмента (A и B) — upstream/вбок от реальной пробки. Поэтому:
1. Сейчас: **Тир 0/1 по сведению** → свести и выпустить «Это другое».
2. Suno-интеграция: быстрый ресёрч статуса API — можно; строить — только не в ущерб релизу.
3. DAW-агент Тир 2-3 и Suno-автоматизацию — после 1–2 выпущенных синглов.
4. FL не бросать (там твой воркфлоу); Reaper — как «движок» для инструмента, не замена.
