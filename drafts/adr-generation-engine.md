# (предложение ADR) — Выбор движка генерации музыки

- **Статус:** 🟡 предложено (НЕ решение на сегодня — материал для будущего ADR)
- **Дата:** 2026-06-21
- **Автор:** rozhkov

## Контекст

Текущий движок — **Suno** (ручной веб-флоу): демка + текст + промпт → каверы → отбор → стемы. По ходу
сессии исследованы: автоматизация Suno (AceData) и легальная альтернатива (ElevenLabs Music).

## Предлагаемое решение (на согласование позже)

- **Suno остаётся творческим движком** — на нём держится метод «cover из демки → AI-сюрприз».
- **AceData (+ SunoMCP)** — путь автоматизации Suno, когда дойдём до масштабирования генерации. Конфиг
  «AceData-only» рабочий (своя Suno-подписка не нужна); цена — pay-as-you-go, мерить на пробных кредитах.
- **ElevenLabs Music API** — легальный/приватный **запасной аэродром**: актуален, только если чистота прав
  станет блокером релиза (ломает метод — нет «cover из демки»).

## Почему пока не решаем

- Генерация — **не текущая пробка** (пробка — сведение; см. [[decisions/0004-ship-before-tooling]]).
- Нужен POC AceData на реальном материале.
- Открыт вопрос коммерческих прав на AI-вывод через неофициального посредника.
