# mistakes/ — журнал ошибок рассуждения агента

Здесь фиксируются **реверсы рассуждения агента**: «рассуждал A → оказалось B → сигнал для будущего».
Это НЕ багтрекер и НЕ список опечаток — только уроки про то, как агент думал неправильно.

## Когда писать запись (порог)

Запись имеет смысл, если выполняется **хотя бы одно**:
1. Ошибка стоила **>30 минут** (своих или чужих).
2. Это **повторяемый паттерн** (тот же тип ошибки уже встречался).
3. **Развернулись после уже сделанной работы** (написали/закоммитили, потом откатили).
4. Урок **поднимается в правило/ADR**.

Ниже порога — обычная строка в `state/daily/`, не отдельный файл.

## Что НЕ писать
Опечатки, синтаксические баги, одноразовое «попробовал — не вышло — попробовал иначе», ошибки
пользователя (это в `decisions/` или daily).

## Формат
Файл `mistakes/YYYY-MM-DD-<slug>.md` по [[_template]]. Обязательное поле — **Signal**
(как распознать ту же ошибку в будущем).

## Жизненный цикл
- Хук `check-mistakes-candidate.ps1` (Stop) ловит кандидатов по pivot-маркерам (`revert`, `оказалось`,
  `развернул`, `откат`…) и `git revert`/`reset --hard`.
- Хук `check-mistakes-daily-review.ps1` (раз в день) напоминает оценить накопленное.
- 2-3 однотипных записи с похожим **Signal** → подними паттерн в `rules/*.md` или новый ADR (не авто-генерь — предложи с черновиком в `drafts/`).
