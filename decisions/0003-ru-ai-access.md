# 0003 — Доступ к ИИ-агенту для участников из РФ

- **Статус:** ✅ принято (финализировано 2026-06-21 по ресёрчу)
- **Дата:** 2026-06-21
- **Автор:** rozhkov

## Контекст

[[team/rozhkov|Свят]] — на корпоративных Claude Code + Cursor. [[team/oleg|Олег]] (РФ, Рязань) подключается
к общей памяти и должен иметь **свой полноценный ИИ-агент в VS Code** (файлы/Git, MCP, YouGile через CLI,
web), **не веб-чат**, оплата им же, рублями, с сохранением `.claude`-хуков (паритет с напарником).
Ресёрч [[drafts/claude-ru-access-risk]]: прямой Claude Pro из РФ = риск **ВЫСОКИЙ** (массбаны ~май 2026 с
удалением данных; РФ — неподдерживаемый регион; условная ID-верификация с 08.07.2026). Ресёрч
[[drafts/claude-code-ru-aggregator-setup]] подтвердил рабочий обходной путь.

## Решение

Участники из РФ используют **Claude Code, направленный на РФ-агрегатор с нативным Anthropic-форматом** —
без аккаунта Anthropic, оплата рублями. Хуки и MCP сохраняются (они клиентские).

- **Основной агрегатор — AITunnel** (нативный Anthropic Messages API, офиц. гайд Claude Code, дешевле, СБП, мин. 399₽).
  Конфиг: `ANTHROPIC_BASE_URL=https://api.aitunnel.ru`, `ANTHROPIC_AUTH_TOKEN=<ключ>`, `ANTHROPIC_API_KEY=""`.
  **В Anthropic не логиниться** (аккаунта нет → персонального бана нет).
- **Fallback-агрегатор — ProxyAPI** (тоже нативный Anthropic + гайд Claude Code) — на случай downtime/бана upstream.
- **Запасной клиент — Cline** (если Claude Code-путь сломается): provider-agnostic, MCP работает, но **`.claude`-хуки теряются**.
- **Модель:** Sonnet 4.6 — рутина, Opus — тяжёлое.
- **Страховка:** vault в **Git** → память не зависит от инструмента/аккаунта.
- Пошаговая настройка — [[howto/oleg-claude-code-setup]].

## Альтернативы

- **Прямой Claude Pro/Codex из РФ** — отвергнуто: HIGH-риск бана с потерей данных.
- **Веб-чат агрегатора** — отвергнуто: не полноценный агент (ручное копирование в Git).
- **OpenAI-only агрегатор + мост** (claude-code-router/LiteLLM) — рабочий план Б, но лишний слой; LiteLLM имел malware-версии. Не нужен, раз AITunnel отдаёт нативный формат.
- **Домашние модели (GigaChat/YandexGPT)** — слабы для агентной работы; только аварийный fallback.

## Последствия

- Олегу: VS Code + Claude Code + ключ AITunnel; `.claude`-хуки и MCP работают как у Свята.
- ⚠️ **Приватность:** агрегатор видит трафик (промпты/контент). Для музыкального vault'а без секретов —
  приемлемо; секреты в базу и так не пишем ([[rules/security]]).
- ⚠️ **Single-point-of-failure:** upstream-аккаунт агрегатора → держим ProxyAPI вторым каналом.
- Не санкционировано Anthropic, гарантий непрерывности нет; наработки безопасны (Git).
- Не использовать схемы «подписка через прокси/OAuth-токен» (Anthropic банил, янв. 2026) — только pay-as-you-go API.
