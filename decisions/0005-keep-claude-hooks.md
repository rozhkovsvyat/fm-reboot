# 0005 — Автоматизация vault'а: остаёмся на Claude-хуках (+ git-страховка)

- **Статус:** ✅ принято
- **Дата:** 2026-06-21
- **Автор:** rozhkov

## Контекст

Развилка: **A** — оставить Claude-хуки и Claude (через AITunnel, см. [[decisions/0003-ru-ai-access]]),
Sonnet по умолчанию, Opus точечно; **B** — переделать дисциплину/автоматизацию под Claude-агностик и уйти
на более дешёвые модели/Codex ради «свободнее». Важное: **память уже портативна** (vault = Git + markdown);
Claude-специфичны только реактивные триггеры (хуки). Бутылочное горло проекта — сведение/релиз, не инфра
([[decisions/0004-ship-before-tooling]]). Профиль Олега — переменный/тяжёлый (task-offload + ресёрч).

## Решение

**Путь A.** Оставляем Claude Code + `.claude`-хуки как механизм автоматизации; Олег — через AITunnel; Sonnet
дефолт, Opus точечно (pay-per-token → тариф не покупаем). **Полный Claude-агностик-ребилд НЕ делаем сейчас.**
Единственный дешёвый шаг устойчивости — **подключить уже существующие git-хуки** (`post-commit.sh` →
`core.hooksPath=_system/githooks/`), чтобы долговременная автоматизация (activity, проверки после pull)
переживала падение Claude/AITunnel и работала даже на Cline. Планировщик / MCP-обёртка / полный rewrite — отложены.

## Альтернативы

- **Путь B (полный Claude-агностик-ребилд + дешёвая модель/Codex)** — отвергнуто СЕЙЧАС: инфра-проект,
  конкурирующий с релизом (против [[decisions/0004-ship-before-tooling]]); не делает тяжёлый ресёрч дешевле
  (тот же аггрегаторный pay-per-token); риск качества на дешёвой модели; теряет per-turn дисциплину-хуки.
  Остаётся **дешёвой будущей опцией** (память портативна → переключение дёшево).

## Последствия

- Near-zero инженерии сейчас; git-хуки — разовая настройка (~1ч, infra → по подтверждению).
- Бюджет Олега переменный ~$20–60+/мес (см. [[team/oleg]]); рычаги цены — модель / лёгкий контекст / prompt-caching.
- **Триггер пересмотра в сторону B:** если AITunnel/Claude на практике ненадёжны (downtime/баны/дорого) —
  эскалируем, дёшево (память уже в Git).
